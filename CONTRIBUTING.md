# Contribuindo

Contribuições são bem-vindas.

Antes de abrir PR:

1. Remova qualquer contexto de projeto privado.
2. Não inclua API keys, logs sensíveis ou dados reais de clientes.
3. Mantenha agentes genéricos e adaptáveis.
4. Explique por que a mudança reduz risco, custo ou confusão operacional.
5. Não transforme a staff em pipeline obrigatória.

Sugestões úteis:

- novos exemplos de roteamento;
- ajustes de idioma;
- alternativas de modelos por provider;
- melhorias de segurança;
- documentação de workflows.
